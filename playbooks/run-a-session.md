# Playbook: Run A Session

## Goal

Run a session that is useful, welcoming, and easy to repeat.

## Before

- Complete the session plan.
- Check the accessibility checklist.
- Prepare shared notes.
- Confirm helpers.
- Test materials.
- Send reminder.

## During

- Start on time.
- State the purpose and level.
- Explain how to ask questions.
- Keep instructions short.
- Pause for questions.
- Capture unresolved issues.
- Close with feedback and next steps.

## After

- Share notes and links.
- Summarise feedback.
- Add resources to the website or repository.
- Thank contributors.
- Invite someone into a small future role.
- Update the shared calendar.

## Common Problems

| Problem | Response |
| --- | --- |
| Too many skill levels in the room | Offer beginner and extension tasks |
| Installation problems consume the session | Provide a cloud option or pair people with helpers |
| One person dominates discussion | Use structured rounds or invite other voices |
| Questions are too specific | Keep an issue list and arrange follow-up |
| Nobody gives feedback | Ask one question before people leave |
