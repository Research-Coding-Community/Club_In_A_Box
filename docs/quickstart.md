# Quickstart

This guide gets a club from idea to first useful session with the least possible ceremony.

## 1. Define The Club

Use the club canvas to answer:

- Who is this for?
- What kind of work will people do together?
- What confidence level should the first sessions assume?
- What is the club not trying to do?
- How often can the organisers realistically run it?

Keep the first version small. A club can grow once people trust that it will meet regularly and respect their time.

## 2. Pick A First Format

Choose one format for the first three sessions:

- Co-working: people bring a problem, work quietly, and ask for help.
- Show-and-tell: two short demos plus discussion.
- Tutorial: a guided exercise with helpers in the room.
- Clinic: people bring bugs, data questions, package problems, or workflow questions.
- Reading group: the group discusses one tool, article, notebook, package, or method.

The first session should make it easy for someone to attend without preparing anything.

## 3. Name The Minimum Team

Avoid making one person responsible for everything. Start with:

- Convenor: keeps the rhythm and makes final calls when needed.
- Facilitator: hosts the session and keeps it moving.
- Helper: supports questions during practical work.
- Notes lead: records links, decisions, and follow-up actions.

One person can hold more than one role at the start, but write the roles down so they can rotate later.

## 4. Create A Basic Home

You need one public place where people can find:

- What the club is for.
- Upcoming sessions.
- How to join.
- Who to contact.
- Notes and resources from past sessions.

For a lightweight club website, see [website guidance](website.md).

## 5. Announce The First Session

Use the event announcement template. Include:

- Date, time, location, and online details if relevant.
- Who the session is for.
- What people should bring.
- Whether beginners are welcome.
- Any accessibility or dietary route.
- Contact details for questions.

## 6. Run The Session

Use the session plan. A reliable first session usually includes:

- Welcome and purpose.
- Quick introductions or check-in.
- Main activity.
- Break or pause.
- Shared notes.
- Feedback and next session topic.

## 7. Follow Up

Within two working days:

- Share notes and useful links.
- Thank contributors.
- Confirm the next session if known.
- Ask one simple feedback question.
- Invite someone to take a small role next time.

## 8. Improve In Public

Treat the club as something the community owns. Keep a visible list of ideas, issues, and improvements. If you adapt this repository, open an issue or pull request here so others can learn from what worked.
