# Governance

This repository uses lightweight governance so it can remain easy to reuse and contribute to.

## Maintainer Responsibilities

Maintainers should:

- Review issues and pull requests.
- Keep contribution routes clear.
- Protect inclusive and accessible language.
- Avoid unnecessary complexity.
- Encourage examples from different settings.
- Make licensing and attribution clear.

## Decision Making

Most changes can be accepted when they are useful, reusable, clearly written, and aligned with the repository principles.

For larger changes, maintainers should consider:

- Does this help more than one club?
- Is it clear how to adapt?
- Does it increase or reduce organiser burden?
- Does it support inclusion and distributed leadership?
- Is there a simpler way to meet the same need?

## Maintainer Changes

New maintainers should have a record of constructive contribution. Maintainers can step back at any time. Where possible, a stepping-back maintainer should help identify a replacement or document outstanding work.

## Community Input

Issues are the preferred route for improvement requests. Pull requests are the preferred route for concrete changes. Discussion should stay focused on making the resource more useful to real clubs.
