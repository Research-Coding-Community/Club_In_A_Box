# Contributing

Contributions are welcome. This repository is intended to be copied, adapted, and improved by the communities that use it.

## Ways To Contribute

- Open an issue for a missing template, unclear guide, accessibility problem, or improvement idea.
- Open a pull request with a small fix.
- Add a new session format or playbook.
- Share an example from your club.
- Improve inclusive language or access guidance.
- Translate or localise materials.
- Report where the guidance is too institution-specific.

## Pull Request Guidance

Before opening a pull request:

- Keep the change focused.
- Use plain language.
- Avoid adding sensitive or identifiable information.
- Prefer reusable templates over institution-only process.
- Add links to examples where useful.
- Check that Markdown renders clearly.

Small pull requests are encouraged. A one-line fix is still a useful contribution.

## Issues

Open an issue when:

- You want a new template.
- You found a confusing instruction.
- Something does not work in your context.
- You want to share how your club adapted the box.
- You have an accessibility or inclusion concern.

## Style

- Write for someone starting from limited context.
- Use short sections and concrete actions.
- Define acronyms.
- Avoid unexplained local policy or platform names.
- Use checklists where they help people act.
